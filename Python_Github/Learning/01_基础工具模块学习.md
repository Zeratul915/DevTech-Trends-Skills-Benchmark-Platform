# 01 基础工具模块学习

## 对应类

- `ConfigManager`
- `LoggerFactory`

## 为什么要有这个模块

基础工具模块不直接处理业务数据，但它让其他模块不用到处写路径、超时时间、线程数和日志格式。这样做的好处是修改配置时只改一个地方。

## 类职责

`ConfigManager` 负责保存项目根目录、数据库文件名、请求超时时间、最大线程数、User-Agent 和默认抓取地址。

`LoggerFactory` 负责统一创建日志对象，保证控制台日志和文件日志格式一致。

## 汇报重点

这两个类体现了低耦合。爬虫、数据库和 Web 模块都可以依赖配置对象，但不用知道配置是怎么组织的。
