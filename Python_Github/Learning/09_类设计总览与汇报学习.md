# 09 类设计总览与汇报学习

## 一句话讲项目

本项目是一个“开发者技术趋势画像与个人技能对标平台”，通过抓取 GitHub Trending 数据，提取技术关键词，分析趋势，并用 Web 页面展示。

## 汇报主线

1. 数据从哪里来：`GithubTrendingSpider` 抓取趋势页面。
2. 数据怎么变干净：`RegexCleaner` 和 `SkillExtractor` 做清洗和关键词提取。
3. 数据怎么保存：`DBManager` 写入 SQLite。
4. 数据怎么变成结论：`Analyzer` 生成排行、词频和技能差距。
5. 结论怎么展示：`Visualizer` 和 Web Controller 生成页面。

## 设计原则

- 单一职责：每个类只负责一个清晰任务。
- 低耦合：模块之间通过数据对象和方法返回值协作。
- 可扩展：以后换数据源、换图表库、换数据库，都只需要修改局部模块。

## 三人分工讲法

- 爬虫负责人罗梓轩：重点讲 `GithubTrendingSpider`、`RepoDetailFetcher`、`RegexCleaner`。
- 后端负责人卢禹辰：重点讲 `DBManager`、`Analyzer`、接口数据流。
- 前端负责人陈载阳：重点讲 `Visualizer`、`DashboardController`、`SkillMatchController`。
