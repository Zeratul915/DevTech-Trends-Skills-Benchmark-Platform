# 11 前后端交互与技能匹配学习

## 对应文件和类

- `app.py`
- `FlaskApp`
- `DashboardController`
- `SkillMatchController`
- `Analyzer.skill_match_report`

## 页面功能

页面会展示当前 GitHub 趋势数据，并让用户勾选自己掌握的技术栈。点击“分析匹配度”后，前端把用户选择发送给后端，后端根据数据库里的技术热度计算匹配结果。

## 前后端交互流程

```text
用户勾选技术栈
        |
        v
前端 JavaScript 收集 checkbox 的 value
        |
        v
POST /api/skill-match
        |
        v
SkillMatchController 调用 Analyzer.skill_match_report
        |
        v
后端返回 JSON：匹配度、短板技能、短板方向、学习建议
        |
        v
前端渲染百分比圆环和建议列表
```

## 后端接口

| 接口 | 方法 | 作用 |
| --- | --- | --- |
| `/` | GET | 打开交互页面 |
| `/api/summary` | GET | 返回仓库数量、语言排行、技能词频等概况 |
| `/api/skills` | GET | 返回可勾选的技术栈 |
| `/api/skill-match` | POST | 根据用户技能计算匹配度和学习建议 |

## 匹配度算法

`Analyzer` 会先根据仓库 star 数给热门技能加权。用户掌握的技能如果出现在热门技能中，就计入已覆盖分数。

```text
匹配度 = 用户已覆盖热门技能分数 / 当前热门技能总分数 * 100
```

然后系统会把用户没覆盖的高热度技能按方向归类，例如前端开发、人工智能、数据库、云原生等，并生成学习建议。

这部分体现了项目不只是静态图表，而是一个能根据用户输入实时分析的交互系统。前端负责收集用户选择和展示结果，后端负责读取趋势数据、计算匹配度、生成学习建议。
