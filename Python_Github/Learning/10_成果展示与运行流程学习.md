# 10 成果展示与运行流程学习

## 对应类

- `TrendPipeline`

## 为什么新增这个类

前面的模块分别负责采集、清洗、存储、分析、可视化和 Web 展示，但汇报时还需要一个“一键跑通”的入口。`TrendPipeline` 就是这个入口，它把所有模块串起来，最后生成 `outputs/dashboard.html`。

## 运行流程

```text
TrendPipeline
    |
    +-- 初始化 ConfigManager 和 DBManager
    +-- 清空旧演示数据，避免重复运行导致数据重复
    +-- 从 GitHub Trending 或 GitHub Search API 获取真实仓库数据
    +-- 使用 SkillExtractor 提取技能关键词
    +-- 使用 DBManager 写入 SQLite
    +-- 使用 Analyzer 统计排行、词频和趋势
    +-- 使用 Visualizer 和 Controller 生成 HTML
    +-- 输出 outputs/dashboard.html
```

## 演示命令

使用示例数据：

```powershell
python run_pipeline.py
```

抓取真实 GitHub 数据：

```powershell
python run_pipeline.py --network --limit 15
```

严格使用真实数据，失败就报错：

```powershell
python run_pipeline.py --network --strict-network --limit 15
```

运行后打开：

```text
outputs/dashboard.html
```

## 汇报重点

`TrendPipeline` 不替代其他模块，而是负责“调度”。它证明项目不是只有类图，而是真的可以把数据从 GitHub 输入，经过数据库和分析模块，最终跑到页面输出。
