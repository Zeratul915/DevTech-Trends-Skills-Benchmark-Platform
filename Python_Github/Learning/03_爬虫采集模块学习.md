# 03 爬虫采集模块学习

## 对应类

- `GithubTrendingSpider`
- `RepoDetailFetcher`

## 类职责

`GithubTrendingSpider` 负责访问 GitHub Trending 页面，并从列表页解析出仓库名称、链接、描述、语言和 star 数。

`RepoDetailFetcher` 负责访问单个仓库详情页，补充 topics 等额外信息。

## 设计原因

列表页抓取和详情页抓取是两种不同任务。拆成两个类以后，列表页解析出错不会影响详情页逻辑，后续也可以只替换其中一个类。

## 汇报重点

Spider 负责“发现数据”，Fetcher 负责“补充数据”。这体现了单一职责原则。
