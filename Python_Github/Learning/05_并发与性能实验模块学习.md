# 05 并发与性能实验模块学习

## 对应类

- `ConcurrentCrawler`
- `PerformanceTester`
- `PerformanceResult`

## 类职责

`ConcurrentCrawler` 提供三种运行方式：单线程、线程池、进程池。

`PerformanceTester` 负责计时和输出对比结果。

`PerformanceResult` 保存每种方式的名称、耗时和处理数量。

## 汇报重点

网络请求属于 IO 密集任务，适合线程池。大量正则清洗或统计可能更接近 CPU 密集任务，可以讨论进程池。

## 演示方式

可以用同一个任务分别跑单线程和线程池，输出耗时对比表，证明项目不是只会调用 API，而是理解了并发使用场景。
