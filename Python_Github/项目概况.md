看了这张课表，我帮你理清一下逻辑：这门课的"暗线"其实是三个阶段——**Stage1(可行性+原型)→Stage2(类设计)→期末(完整可运行系统+演示)**，而"明线"教学内容依次覆盖了 Python基础→函数模块文件→正则/网络/数据库/多线程多进程→Web基础→数据分析可视化→Scrapy爬虫。所以大作业必须是一个**能把这些知识点串成一条完整数据管道的项目**，而不是零散的小练习拼凑。

## 选题：「开发者技术趋势画像与个人技能对标平台」

一句话定位：**定期抓取 GitHub Trending / 技术社区热门内容，清洗成结构化技能数据，做趋势分析和可视化，并让用户输入自己的技能生成"雷达图对比+差距建议"，最后用网页展示。**

选这个而不是抓招聘网站，是因为 GitHub Trending、公开技术社区这类目标**合规风险低、反爬没那么变态**，适合课程周期内稳定完成；如果你们组更想做招聘数据，也可以，但一定要检查目标站 robots.txt、控制请求频率，我建议课上先跟老师确认允许的抓取对象。

### 新意点（体现在演示里会很加分）

1. **时间维度**：不是抓一次就完事，而是每次上课增量抓取一次，攒出真实的趋势折线，而不是伪造数据。
2. **个人技能雷达图对比**：用户勾选自己会的技术栈，系统对比行业热度差距，给出"建议学习方向"，把纯数据项目变成有实用价值的小工具。
3. **性能对比实验**：报告里放一张"单线程 vs 线程池 vs 进程池"耗时对比图，直接对应课程的多线程/多进程难点，老师一眼能看出你们理解了这个知识点而不是背了API。
